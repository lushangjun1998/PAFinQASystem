# PAFinQASystem
基于 `RAG + LLM` 的金融数据问答系统。项目面向结构化金融数据问答场景，完成了两个核心任务：

1. 从自然语言问题中识别并匹配数据库中的指标维度原词，用时控制在 `2s` 内，最好 `1s` 内。
2. 基于识别结果生成可执行的 SQLite SQL 语句，耗时控制在 `10s` 内，时间越短越好。


## 一、整体方案

本项目采用两阶段方案：

1. `字段/表召回阶段`
   根据问题从字段知识库中召回最相关的字段、表和上下文信息。
2. `SQL 生成阶段`
   将问题、召回到的字段信息和数据库 schema 一起输入大模型，生成 SQLite SQL。

系统流程如下：
```text
用户问题 ——> 字段知识库/表结构知识 ——> 混合召回 ——> 相关字段/相关表/上下文 ——>  Prompt 组装
              |--> FAISS  稠密检索                                               ⬇
              |--> BM25 关键词检索                                          LLM 生成 SQL 
                                                                                ⬇
                                                                           SQLite 执行校验
                                                                                ⬇
                                                                             输出结果


```

## 四、下一步建议
### 1. 增加 SQL 自校验与自动修复链路
针对生成错误SQL情况，可以在 SQL 生成后增加一层：

1. 先执行 SQL
2. 如果报错，则把报错信息回传给 LLM
3. 让模型基于报错信息重新修复 SQL

这样通常能显著提升最终可执行率，尤其适合中文字段名和复杂关联场景。

### 2. 增强字段知识库建设

可以继续扩充 `field_knowledge.json` 中的内容，例如：

- 更多同义词
- 常见问题模板
- 时间表达归一化规则
- 金融领域术语别名

字段知识越完整，任务1召回越稳，任务2 SQL 生成也会更准确。

### 3. 引入结果级评估与错误分析看板

给出每个sql的正确结果，并增加一份错误分析文档或可视化统计，例如：

- 哪类问题最容易出错
- 哪些字段最容易混淆
- 哪些 SQL 模式失败率最高
